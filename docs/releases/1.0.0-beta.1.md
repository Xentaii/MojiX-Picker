# MojiX 1.0.0-beta.1

> **Major beta release. Migration is required.**

This beta collects the 1.0 migration work and the latest package-delivery
changes into the first public prerelease for the new package shape. It is meant
for integration testing before promoting the line to stable.

Publish this prerelease to npm with the `beta` dist-tag, not `latest`.

## Highlights

- CDN-first unicode emoji data loading via `loadEmojiData()`
- offline bootstrap via `mojix-picker/data`
- explicit locale registration via `mojix-picker/locales/<code>` and `registerEmojiLocalePack(...)`
- lazy locale search indexes via `mojix-picker/locales/<code>/search`
- vendor sprite preset subpaths via `mojix-picker/sprites/<vendor>`
- headless-only entry via `mojix-picker/headless`
- lazy virtualized grid chunk for large emoji lists
- precompressed `.br` and `.gz` package assets for self-hosted mirrors

## Breaking changes

- `getUnicodeEmojiData()` now throws until data has been loaded or preloaded
- `emojiPickerLocales` starts empty and reflects only explicitly registered locale packs
- CommonJS builds were removed
- old side-effect locale-import patterns should be replaced with explicit registration
- generated `emoji-data.json` no longer carries English names or vendor availability
- exact runtime English names now come from the English locale pack
- sprite vendor availability is loaded from sidecar files by the vendor presets

## Package shape

The package ships `dist/lib/` plus `dist/data/`. The main entry remains code
only; `dist/data/` powers jsDelivr delivery, self-hosted mirrors, and explicit
offline imports.

Current dry-run package size:

- tarball: about `1.44 MB`
- unpacked: about `2.93 MB`
- files: `184`
