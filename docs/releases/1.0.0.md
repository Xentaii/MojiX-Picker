# MojiX 1.0.0

> **Major release. Migration is required.**

MojiX 1.0 moves the package to a CDN-first data model and an ESM-only publish
surface. The default picker now installs smaller and boots without shipping the
unicode emoji dataset inside the main entry, while offline consumers get
explicit preload modules instead of hidden side effects.

Package size improved from about `1.0 MB` to about `505 KB` tarball and from
about `5.7 MB` to about `2.67 MB` unpacked.

## Highlights

- CDN-first unicode emoji data loading via `loadEmojiData()`
- offline bootstrap via `mojix-picker/data`
- explicit locale registration via `mojix-picker/locales/<code>` and `registerEmojiLocalePack(...)`
- vendor sprite preset subpaths via `mojix-picker/sprites/<vendor>`
- ESM-only package output
- updated docs, migration guide, and release workflow

## Breaking changes

- `getUnicodeEmojiData()` now throws until data has been loaded or preloaded
- `emojiPickerLocales` starts empty and reflects only explicitly registered locale packs
- CommonJS builds were removed
- old side-effect locale-import patterns should be replaced with explicit registration

## Notes

This release focuses on package shape, runtime loading behavior, and long-term
maintainability. If your app uses only the top-level picker with modern ESM
tooling, the migration is usually straightforward. If you rely on engine-level
data access, locale side effects, or CommonJS, plan the upgrade carefully.
