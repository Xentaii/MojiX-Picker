# MojiX 1.0.0-beta.2

> **Beta release. Migration is required for apps upgrading from pre-1.0
> versions.**

This beta focuses on runtime smoothness, WebView2/Tauri validation, and the
last round of package/readiness work before promoting the 1.0 line further.
Publish this prerelease to npm with the `beta` dist-tag, not `latest`.

## Highlights

- prepared emoji data cache with idle chunking and an optional Worker pipeline
- lazy category shard loading with scroll-velocity adaptive overscan
- sprite style caching, warmed sprite sheet retention, and shared decoded URLs
- smoother Windows WebView2/Tauri scrolling with an explicit emoji font stack
  and reduced hover-preview work during active scroll
- Tauri React fixture that consumes the packed local npm artifact with local
  data, locale packs, and local sprite sheets
- Tauri profiling path without Vite HMR websocket or forward-console noise
- refreshed responsive demo layout and expanded API/CDN test coverage
- Node 24-aligned CI/release workflows and corrected npm provenance metadata

## Upgrade notes

There are no new intended breaking package-shape changes relative to
`1.0.0-beta.1`. Apps upgrading from `0.5.x` or earlier should still follow the
1.0 migration guide because the beta line keeps the CDN-first data model,
explicit locale registration, ESM-only package surface, and offline data
subpaths introduced in `1.0.0-beta.1`.

For Tauri or other native WebView consumers, the recommended path is local data
plus local sprite sheets. The included fixture demonstrates that setup and keeps
runtime network fetches disabled.

## Package shape

The package ships `dist/lib/` plus `dist/data/`. The main entry remains code
only; `dist/data/` powers jsDelivr delivery, self-hosted mirrors, and explicit
offline imports.

Current dry-run package size:

- tarball: about `1.7 MB`
- unpacked: about `3.6 MB`
- files: `218`
